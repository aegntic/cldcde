# FPEF Framework - Find-Prove-Evidence-Fix

Systematic framework for analyzing, validating, and intervening in complex systems with evidence-based decision making and minimal intervention principles.

## FPEF Process Framework

### Phase 1: FIND - System Mapping
- **Architecture Discovery** - Map system components and relationships
- **Issue Identification** - Locate pain points and bottlenecks
- **Stakeholder Analysis** - Identify all affected parties
- **Data Collection** - Gather comprehensive system data

### Phase 2: PROVE - Causal Analysis
- **Evidence Verification** - Validate assumptions with data
- **Root Cause Analysis** - Trace issues to fundamental causes
- **Impact Assessment** - Measure effects and dependencies
- **Risk Evaluation** - Assess intervention risks

### Phase 3: EVIDENCE - Data Validation
- **Statistical Analysis** - Quantitative evidence gathering
- **Qualitative Review** - Context understanding
- **Peer Review** - External validation
- **Documentation** - Complete evidence trail

### Phase 4: FIX - Minimal Intervention
- **Targeted Solutions** - Precise intervention design
- **Impact Measurement** - Real-time effectiveness tracking
- **Rollback Planning** - Safe failure recovery
- **Continuous Optimization** - Iterative improvement

## Usage Examples

**System Performance Investigation:**
```
/fpef framework map "Database performance degradation"
```

**Evidence-Based Fix Implementation:**
```
/fpef framework verify "Memory leak in Node.js application"
```

**Complete Analysis Cycle:**
```
/fpef framework all "User authentication system failures"
```

## Output Structure

Each FPEF cycle generates:
- **System Map** with component relationships
- **Evidence Report** with validation data
- **Causal Analysis** with root cause identification
- **Intervention Plan** with minimal fixes
- **Risk Assessment** with mitigation strategies
- **Monitoring Framework** for ongoing validation

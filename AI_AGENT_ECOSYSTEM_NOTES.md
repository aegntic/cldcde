## Overview: Agentic Engineering Frameworks (2025/2026)

The notebook covers the shift from "chatting with code" to **Agentic Engineering**—where humans provide **Spec and Context**, and Agents handle **Implementation and Verification**.

## 5 High-Leverage Prompt Frameworks

### 1. The "Spec-First" Architect Protocol

**Problem:** Agents fail when they jump straight to coding ("vibe coding" creates unmaintainable code)

**Solution:** Force a "Thinking Phase" using RDoLT (Recursive Decomposition of Logical Thoughts) to separate planning from execution.

```
Act as a Principal Software Architect.
TASK: Create a comprehensive implementation spec for [FEATURE_NAME].
DO NOT write code yet.

STEP 1: RECURSIVE DECOMPOSITION (Level 1 -> 3)
1. Easy: List the high-level components and file structure changes.
2. Intermediate: Define the data interfaces, API signatures, and state management flow.
3. Final: Identify edge cases, security risks (e.g., input validation, auth), and potential breaking changes.

STEP 2: THE BLUEPRINT
Generate a file named `SPEC-[FEATURE].md` containing:
- [ ] Goal & Constraints
- [ ] New/Modified File List
- [ ] Step-by-Step Execution Plan (atomic steps an agent can follow)
- [ ] Testing Strategy (unit vs. integration)

Wait for my approval of `SPEC.md` before writing a single line of code.
```

### 2. The "Zero-Hallucination" Context Bridge

**Problem:** Feeding raw docs to an agent bloats context and causes hallucinations

**Solution:** Use NotebookLM as a grounded knowledge layer (RAG pipeline)

**Setup:**

1. Upload docs/SDKs to Google NotebookLM
2. Create a SKILL.md or use NotebookLM MCP

```
I am building [FEATURE] using [LIBRARY/FRAMEWORK].
STOP. Do not guess the API methods.

ACTION:
1. Query my 'Docs-Notebook' via the NotebookLM tool/skill.
2. Ask: "What are the exact API methods, required parameters, and authentication flows for [SPECIFIC_TASK] in [LIBRARY]?"
3. Retrieve the cited code examples.
4. ONLY after you have the cited source, generate the implementation code.

If you cannot find the answer in the Notebook, STOP and ask me for the documentation.
```

### 3. The "Ralph Loop" (Self-Healing Code)

**Problem:** Agents write code that looks right but fails at runtime

**Solution:** Encode a Feedback Loop (Write → Test → Fix)

```
/ultrawork "Implement [TASK] with a Self-Healing Loop"

PROTOCOL:
1. IMPLEMENT: Write the code for [TASK] in @src/.
2. VERIFY: Immediately create and run a test script `test_task.py` (or use existing tests).
3. LOOP:
   - If tests PASS: Delete the test script and commit with message "feat: [TASK]".
   - If tests FAIL: Read the error log. Do not ask me. Reason about the root cause. Apply a fix. Retry the test.
   - Max Retries: 3. If it still fails, restore original state and report the blocker.

Start the loop.
```

### 4. The "Agent Handoff" (Meta-Context)

**Problem:** You switch agents and lose context, or the agent "forgets" the rules

**Solution:** Create a persistent `AGENTS.md` or `CLAUDE.md` file as long-term memory

```
Analyze this entire repository structure @./.
Generate a `AGENTS.md` file that teaches ANY future AI agent how to work here.

Include:
1. **The Stack**: Languages, frameworks, and key libraries.
2. **The Rules**: No 'any' types, use 'feature-first' folder structure, specific linter rules.
3. **The Workflow**: "Always run `npm test` before committing."
4. **The "Do Not" List**: Common mistakes you have observed (e.g., "Do not use deprecated API X").

This file will be your system prompt for all future sessions.
```

### 5. The "Hybrid Team" Orchestration

**Problem:** One model isn't good at everything

**Solution:** Route tasks to the best model (Gemini 3 = creative/visual, Claude Opus = logical/strict)

**Workflow:**

**Step 1 (Gemini 3 / Antigravity - The Designer):**

```
Act as a Product Designer. Review the `SPEC.md`.
Generate the Frontend Code (React/Tailwind) for [COMPONENT].
Focus on: Visual hierarchy, responsive layouts, and user delight.
Do not worry about the complex backend logic yet.
```

**Step 2 (Claude Code / Opus 4.5 - The Engineer):**

```
Act as a Backend Engineer.
Take the frontend component @[Component.tsx] generated by the designer.
1. Wire it to the real API endpoints defined in `SPEC.md`.
2. Add strict error handling and loading states.
3. Refactor any sloppy code to match our `AGENTS.md` standards.
```

## ðŸ”„ THREE-PHASE EXECUTION PROTOCOL

### Phase 1: System Mapping (MANDATORY FIRST STEP)
**Before ANY action, answer these questions:**

#### Critical Questions
- **What is the user's PRECISE outcome?** (Rephrase in your own words to confirm)
- **What is the system CURRENTLY doing?** (Gather evidence ONLY)
- **Where is the root disconnect?** (Trace expected vs actual behavior)
- **What are ALL components and boundaries?**

#### Inner Monologue
> **"STOP. Map reality. Evidence only - no docs, no patterns, no assumptions."**

#### Implementation Requirements
- **No Solutions Proposed**: Until system mapping is complete
- **Evidence Gathering Only**: Use commands, logs, tests, configurations
- **Component Mapping**: Identify all system parts and their relationships
- **Boundary Definition**: Clear understanding of system scope and limits

### Phase 2: Evidence-Driven Verification (MANDATORY SECOND STEP)
**Test EVERY assumption with concrete, reproducible actions:**

#### Verification Process
- **Build Mental Model from FACTS**: Use logs, tests, configurations only
- **Document Observations**: "curl shows 401; logs confirm basic auth enabled"
- **No Unverified Beliefs**: Every assumption must be tested and proven
- **Reproducible Actions**: Others should be able to replicate your verification

#### Inner Monologue
> **"Verify now. Does this match reality? Test before trust."**

#### Implementation Requirements
- **Concrete Testing**: Use actual commands and tools
- **Observable Results**: Document what actually happens
- **Evidence Documentation**: Record all observations and test results
- **Assumption Challenge**: Question every belief and verify it

### Phase 3: Minimal Viable Intervention (MANDATORY THIRD STEP)
**Identify the SIMPLEST change that bridges the gap:**

#### Intervention Strategy
- **Deliver Complete Working Result**: Not partial solutions
- **Include Verification Proof**: Demonstrate that it works
- **Confirm Outcome Achievement**: "This achieves [user goal] - verified by [test]"

#### Inner Monologue
> **"Minimal change? Outcome achieved? Test the result, not the parts."**

#### Implementation Requirements
- **Minimal Change Principle**: Smallest change that achieves the outcome
- **Complete Solution**: Fully functional, not theoretical
- **Verification Included**: Proof that the solution works
- **Outcome Confirmation**: Clear statement of goal achievement
